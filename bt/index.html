BLE 广播数据协议定义

本小节定义了用于 BLE 广播包的自定义数据格式。此格式适用于指定的公司标识符、产品标识符、设备电量信息、经典蓝牙地址，以及软件版本号的传输。

1. 广播数据格式

广播数据的结构如下所示：

xxxxxxxxxx
    uint8_t adv_data[] = {
        0x14,                   // 长度字段 (20 个字节，包括类型和数据)
        0xFF,                   // 类型字段 (0xFF 表示 Manufacturer Specific Data)
        0x0E, 0x00,             // 公司标识符 (Company ID=0x000E，按 BLE 规范小端存放 => 0x0E 0x00)
        0x53, 0x01,             // 产品标识符 S(0x53)/T(0x54)：ASCII 字符， 1(0x01):Int,
        0x64,                   // 左耳电量 (100%)
        0x50,                   // 右耳电量 (80%)
        0x32,                   // 充电仓电量 (50%)
        0x01, 0x23, 0x45, 0x67, 0x89, 0xAB, // 经典蓝牙地址 (01:23:45:67:89:AB): hexString是蓝牙地址
    0x01,                   // 充电仓颜色（0黑，1白，2紫，3粉，4金，5透明灰，6透明白，7蓝色，8灰色）
    0x31, 0x2E, 0x30, 0x2E, 0x30 // 软件版本号 "1.0.0" (ASCII 字符 1.0.0)
};

2. 字段说明

  • 长度字段（1 字节）：0x14

    • 表示后续数据的总字节数，包括类型字段、公司标识符、产品标识符、电量信息、经典蓝牙地址和软件版本号。

    • 长度值为 20，表示广播包中的数据总长度为 20 字节(类型和数据)。

  • 类型字段（1 字节）：0xFF

    • 固定值 0xFF，表示此字段为 Manufacturer Specific Data 类型。

  • 公司标识符（2 字节）：0x0E, 0x00

    • 由 Bluetooth SIG 分配的 Company Identifier Code：0x000E。

    • 在 Manufacturer Specific Data 中按 BLE 规范小端序存放，因此字节序列为 0x0E, 0x00（而不是写成 0x00, 0x0E）。

  • 产品标识符（2 字节）：0x53, 0x01 或 0x54, 0x01

    • 第一个字节

      ：产品类别标识符。

      • 0x53 表示 S 型设备,即 SOUNDPEATS。

      • 0x54 表示 T 型设备,即 TRUEFREE。

    • 第二个字节：产品型号标识符，用于进一步区分同类别中的不同产品。

  • 左耳电量（1 字节）：0x64

    • 表示左耳设备的电池电量，单位为百分比。取值范围为 0x00（0%）到 0x64（100%）。

  • 右耳电量（1 字节）：0x50

    • 表示右耳设备的电池电量，单位为百分比。取值范围为 0x00（0%）到 0x64（100%）。

  • 充电仓电量（1 字节）：0x32

    • 表示充电仓的电池电量，单位为百分比。取值范围为 0x00（0%）到 0x64（100%）。

  • 经典蓝牙地址（6 字节）：0x01, 0x23, 0x45, 0x67, 0x89, 0xAB

    • 经典蓝牙设备的 MAC 地址，采用标准的 6 字节格式。

    • 字节顺序：按 MAC 地址字符串从左到右存放，例如 01:23:45:67:89:AB 对应 01 23 45 67 89 AB。

    • 如芯片/协议栈提供的地址字节序与上述相反（常见于某些接口按 LSB-first 提供），固件需在写入广播数据前做字节序转换；App 端按本顺序解析。

  • 耳机颜色（1 字节）：0x01

    • 表示耳机的颜色

      • 0x00 表示 黑色

      • 0x01 表示 白色

      • 0x02 表示 紫色

      • 0x03 表示 粉色

      • 0x04 表示 金色/米黄色

      • 0x05 表示 透明灰

      • 0x06 表示 透白色

      • 0x07 表示 蓝色

      • 0x08 表示 灰色

      • 0x09 表示 米黄色

  • 软件版本号（5 字节）：0x31, 0x2E, 0x30, 0x2E, 0x30

    • 表示软件版本号

      xxxxxxxxxx
      1.0.0

      ，每个字节为对应的 ASCII 字符：

      • 0x31 表示字符 1

      • 0x2E 表示字符 .

      • 0x30 表示字符 0

4. 解析流程

对接方案商可以通过以下流程解析广播包中的数据：

  1. 识别公司标识符：

    • 从广播包中解析公司标识符字段，确保字节序列为 0x0E, 0x00（Company ID=0x000E）。

  2. 识别产品标识符：

    ⭐️ 由星科启为每个型号的耳机分配唯一标识，应该与获取固件版本处出现的产品标识符保持一致

    • 根据第一个字节区分产品类别（0x53 表示 S 型设备，0x54 表示 T 型设备）ASCII 编码的字符。

    • 第二个字节表示产品的具体型号（ Int 类型）。

  3. 获取电量信息：

    • 左耳电量、右耳电量和充电仓电量均为 1 字节字段，表示电量百分比。

  4. 获取经典蓝牙地址：

    • 解析 6 字节的经典蓝牙地址，按照 MAC 地址格式显示。

  5. 获取耳机颜色

    • 解析 1 字节的耳机颜色

  6. 获取软件版本号：

    • 解析 5 字节的 ASCII 字符串，组合成完整的版本号。例如，0x31, 0x2E, 0x30, 0x2E, 0x30 解析为 1.0.0。

5. 注意事项

  • 兼容性：广播包设计需符合 BLE 规范，确保兼容性和互操作性。

  • 扩展性：如需扩展广播数据，可以增加自定义字段，但需确保长度字段相应更新。

  • 安全性：确保广播数据的安全性，避免泄露敏感信息，如经典蓝牙地址的加密或保护。

此协议旨在确保 BLE 广播数据的可靠传输和解析，请各方严格按照本协议实现对接，以确保系统的正常运作。

数据包注释

位置（上面示例设置降噪模式为ANC）：

  • SOF

    起始头：0xFF

  • Version 协议版本: 0x04

  • Flags 标志， 0x00

  • Length PAYLOAD 的长度：

    • 以Optional PAYLOAD参数的实际长度为准(重要)，例如图中 Optional PAYLOAD的参数是01,则长度为01

  • 字节序（Endianness）

    • 除特别说明外，本协议中所有多字节字段均为高字节在前（Big-Endian），例如 VENDOR ID=0x000A 对应字节序列 00 0A，COMMAND=0x0311 对应 03 11。

  • VENDOR ID

    厂商 ID：

    • 非EQ数据：0x000a，

    • EQ数据：0x001d

  • COMMAND 功能指令 0x03 11（返回时 0x83 11）

  • Optional PAYLOAD 可选的参数 例如 降噪功能:

    • 普通模式 0x00

    • ANC 模式 0x01

    • 通透模式 0x02

    • 环境自适应 0x03

    • ANC 模式（弱） 0x04

获取固件版本

⭐️产品标识符_打包时间_版本号 （版本号需要与打包出来的OTA 升级文件名上的后三部分保持一致）

  1. 指令：0x03 09

  2. Payload：无

  3. 返回: 版本号返回相应的版本字符的 ASCII 码，格式:产品标识符_编译日期_v版本号，
    如: S8_20201030_v1.0.0

    版本格式

    版本号采用语义化版本的格式，形式为：主版本号.次版本号.修订号 。

    版本号递增规则

    • 主版本号：当进行了不兼容的 API 修改时递增。

    • 次版本号：当新增了向下兼容的功能时递增。

    • 修订号：当进行了向下兼容的问题修正时递增。

    示例

    • 1.0.0：初始发布版本。

    • 1.1.0：增加了新功能，并保持向下兼容。

    • 1.1.1：修复了现有功能中的问题，并保持向下兼容。

    • 2.0.0：进行了不兼容的 API 修改。

    通过这种方式，版本号能清晰地反映软件的更新类型和兼容性变化，帮助用户和开发者更好地理解版本更新的内容。

  4. 示例

    0xff 04 00 18 00 0a 83 09 `00 42 53 33 39 30 52 42 5f 32 30 32 31 31 32 30 36 5f 76 30 2e 33 2e 39`
